# Sequence Diagram — P2 Sub-Processes 2.4 & 2.5
## Validate & Bind JIRA / GitHub

> Endpoints: `POST /api/groups/{groupId}/jira`, `POST /api/groups/{groupId}/github`
> Issues: B-06, B-10, B-11, B-12, B-14
> JWT principal = internal student UUID
> Both flows are structurally similar — JIRA shown in full, GitHub shown with diffs only.
> **Implementation note:** Both bind methods support re-bind (token rotation) — on re-bind, credentials are overwritten but group status is preserved.

---

### POST /api/groups/{groupId}/jira (Bind JIRA)

```mermaid
sequenceDiagram
    actor Student as Student (Team Leader)
    participant GroupController
    participant GroupService
    participant GroupMembershipRepository
    participant JiraValidationService
    participant JiraAPI as JIRA REST API
    participant EncryptionService
    participant ProjectGroupRepository
    participant AuditLogService

    Student->>GroupController: POST /api/groups/{groupId}/jira<br/>{jiraSpaceUrl, jiraEmail, jiraProjectKey, jiraApiToken, jiraTokenExpiresAt}<br/>Authorization: Bearer <student_jwt>

    GroupController->>GroupController: extract requesterUUID from SecurityContext
    GroupController->>GroupService: bindJira(groupId, jiraSpaceUrl, jiraEmail, jiraProjectKey, jiraApiToken, jiraTokenExpiresAt, requesterUUID)

    GroupService->>GroupMembershipRepository: findByGroupIdAndStudentId(groupId, requesterUUID)
    GroupMembershipRepository-->>GroupService: Optional<GroupMembership>

    alt not TEAM_LEADER
        GroupService-->>GroupController: throw ForbiddenException
        GroupController-->>Student: 403 {"error": "Only the Team Leader can bind tool integrations"}
    end

    Note over GroupService: DISBANDED freeze — fetch group before any external call
    GroupService->>ProjectGroupRepository: findById(groupId)
    ProjectGroupRepository-->>GroupService: ProjectGroup

    alt group.status == DISBANDED
        GroupService-->>GroupController: throw BusinessRuleException
        GroupController-->>Student: 400 {"error": "This group has been disbanded"}
    end

    Note over GroupService,JiraAPI: Live validation — nothing is stored until this passes
    GroupService->>JiraValidationService: validate(jiraSpaceUrl, jiraEmail, jiraProjectKey, jiraApiToken)
    JiraValidationService->>JiraAPI: GET {jiraSpaceUrl}/rest/api/3/project/{jiraProjectKey}<br/>Authorization: Basic base64(jiraEmail:jiraApiToken)<br/>timeout: 5s

    alt 401 / 403
        JiraAPI-->>JiraValidationService: 401/403
        JiraValidationService-->>GroupService: throw JiraValidationException
        GroupService->>AuditLogService: record(requesterUUID, STUDENT, JIRA_BOUND, GROUP, FAILURE)
        GroupService-->>GroupController: propagate
        GroupController-->>Student: 422 {"error": "JIRA validation failed: API token is invalid or expired"}
    else 404
        JiraAPI-->>JiraValidationService: 404
        JiraValidationService-->>GroupService: throw JiraValidationException
        GroupController-->>Student: 422 {"error": "JIRA validation failed: Project key '{key}' not found"}
    else timeout / network error
        JiraAPI-->>JiraValidationService: timeout
        JiraValidationService-->>GroupService: throw JiraValidationException
        GroupController-->>Student: 422 {"error": "JIRA validation failed: JIRA space URL is unreachable"}
    else 200 OK
        JiraAPI-->>JiraValidationService: 200
        JiraValidationService-->>GroupService: return (no exception)
    end

    GroupService->>EncryptionService: encrypt(jiraApiToken)
    EncryptionService-->>GroupService: encryptedToken (Base64Url[IV || ciphertext || authTag])

    Note over GroupService: Determine new status:<br/>- Re-bind (token already exists) → keep current status<br/>- First-time bind + GitHub already bound → TOOLS_BOUND<br/>- First-time bind + GitHub not bound + FORMING → TOOLS_PENDING<br/>- Otherwise keep current status
    GroupService->>GroupService: isRebind = encryptedJiraToken != null
    GroupService->>GroupService: newStatus = isRebind ? currentStatus : (githubBound ? TOOLS_BOUND : TOOLS_PENDING)

    GroupService->>ProjectGroupRepository: save(group{jiraSpaceUrl, jiraEmail, jiraProjectKey, encryptedJiraToken, jiraTokenValid=true, jiraTokenExpiresAt, status=newStatus})
    ProjectGroupRepository-->>GroupService: savedGroup

    GroupService->>AuditLogService: record(requesterUUID, STUDENT, JIRA_BOUND, GROUP, SUCCESS)

    GroupService-->>GroupController: BindToolResponse
    GroupController-->>Student: 200 {groupId, status, jiraSpaceUrl, jiraProjectKey, jiraBound:true, githubBound}
```

### POST /api/groups/{groupId}/github (Bind GitHub)

```mermaid
sequenceDiagram
    actor Student as Student (Team Leader)
    participant GroupController
    participant GroupService
    participant GroupMembershipRepository
    participant GitHubValidationService
    participant GitHubAPI as GitHub REST API
    participant EncryptionService
    participant ProjectGroupRepository
    participant AuditLogService

    Student->>GroupController: POST /api/groups/{groupId}/github<br/>{githubOrgName, githubPat, githubRepoName}<br/>Authorization: Bearer <student_jwt>

    GroupController->>GroupController: extract requesterUUID from SecurityContext
    GroupController->>GroupService: bindGitHub(groupId, githubOrgName, githubPat, githubRepoName, requesterUUID)

    GroupService->>GroupMembershipRepository: findByGroupIdAndStudentId(groupId, requesterUUID)
    GroupMembershipRepository-->>GroupService: Optional<GroupMembership>

    alt not TEAM_LEADER
        GroupService-->>GroupController: throw ForbiddenException
        GroupController-->>Student: 403 {"error": "Only the Team Leader can bind tool integrations"}
    end

    Note over GroupService: DISBANDED freeze — fetch group before any external call
    GroupService->>ProjectGroupRepository: findById(groupId)
    ProjectGroupRepository-->>GroupService: ProjectGroup

    alt group.status == DISBANDED
        GroupService-->>GroupController: throw BusinessRuleException
        GroupController-->>Student: 400 {"error": "This group has been disbanded"}
    end

    Note over GroupService,GitHubAPI: Sequential calls — each subsequent call is skipped if prior fails
    GroupService->>GitHubValidationService: validate(githubOrgName, githubPat, githubRepoName)

    GitHubValidationService->>GitHubAPI: GET /orgs/{githubOrgName}<br/>Authorization: Bearer {githubPat}<br/>timeout: 5s

    alt 401
        GitHubAPI-->>GitHubValidationService: 401
        GitHubValidationService-->>GroupService: throw GitHubValidationException
        GroupService->>AuditLogService: record(requesterUUID, STUDENT, GITHUB_BOUND, GROUP, FAILURE)
        GroupController-->>Student: 422 {"error": "GitHub validation failed: PAT is invalid or expired"}
    else 404
        GitHubAPI-->>GitHubValidationService: 404
        GitHubValidationService-->>GroupService: throw GitHubValidationException
        GroupController-->>Student: 422 {"error": "GitHub validation failed: Organization '{name}' not found"}
    else 200 OK — org found, PAT valid
        GitHubAPI-->>GitHubValidationService: 200

        GitHubValidationService->>GitHubAPI: GET /orgs/{githubOrgName}/repos?per_page=1<br/>Authorization: Bearer {githubPat}

        alt 403 — missing repo scope
            GitHubAPI-->>GitHubValidationService: 403
            GitHubValidationService-->>GroupService: throw GitHubValidationException
            GroupController-->>Student: 422 {"error": "GitHub validation failed: PAT lacks required 'repo' scope"}
        else 200 OK — scope verified
            GitHubAPI-->>GitHubValidationService: 200

            Note over GitHubValidationService: Repo existence check
            GitHubValidationService->>GitHubAPI: GET /repos/{githubOrgName}/{githubRepoName}<br/>Authorization: Bearer {githubPat}

            alt 404 — repo not found
                GitHubAPI-->>GitHubValidationService: 404
                GitHubValidationService-->>GroupService: throw GitHubValidationException
                GroupController-->>Student: 422 {"error": "GitHub validation failed: Repository not found"}
            else 200 OK — returns tokenExpiresAt from response headers
                GitHubAPI-->>GitHubValidationService: 200
                GitHubValidationService-->>GroupService: GitHubValidationResult{tokenExpiresAt}
            end
        end
    end

    GroupService->>EncryptionService: encrypt(githubPat)
    EncryptionService-->>GroupService: encryptedPat

    Note over GroupService: Determine new status:<br/>- Re-bind (PAT already exists) → keep current status<br/>- First-time bind + JIRA already bound → TOOLS_BOUND<br/>- First-time bind + JIRA not bound → TOOLS_PENDING
    GroupService->>GroupService: isRebind = encryptedGithubPat != null
    GroupService->>GroupService: newStatus = isRebind ? currentStatus : (jiraBound ? TOOLS_BOUND : TOOLS_PENDING)

    GroupService->>ProjectGroupRepository: save(group{githubOrgName, githubRepoName, encryptedGithubPat, githubTokenValid=true, githubPatExpiresAt, status=newStatus})
    ProjectGroupRepository-->>GroupService: savedGroup

    GroupService->>AuditLogService: record(requesterUUID, STUDENT, GITHUB_BOUND, GROUP, SUCCESS)

    GroupService-->>GroupController: BindToolResponse
    GroupController-->>Student: 200 {groupId, status, githubOrgName, githubBound:true, jiraBound}
```
